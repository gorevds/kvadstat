# Избранное (favorites) — дизайн

Дата: 2026-06-18

## Цель

Дать пользователю отмечать понравившиеся квартиры и возвращаться к ним:
- кнопка «в избранное» в боковой панели деталей (там же, где открывается планировка);
- отдельная вкладка «⭐ Избранное» со списком отмеченных;
- мягкая подсветка избранных строк в общем списке «Все объекты».

Сайт публичный, без авторизации, поэтому избранное — **per-browser** (localStorage),
не синхронизируется между устройствами. Для публичной витрины это приемлемо.

## Объём

В объёме:
- localStorage-хранилище множества id квартир;
- toggle-кнопка в панели деталей (index.html);
- класс-подсветка избранных строк в таблице «Все объекты»;
- 4-я вкладка «⭐ Избранное» с таблицей (те же колонки, что «Все объекты»);
- в избранном по умолчанию — только активные (есть в текущем срезе today_all),
  плюс кнопка-тумблер «Показать снятые с продажи», подтягивающая снятые
  из истории и показывающая их приглушённо с пометкой и датой последнего среза;
- кросс-вкладочная синхронизация (событие `storage`);
- backend: canned-query `fav_flats` в metadata.yml для снятых с продажи.

Вне объёма (YAGNI): серверное/синхронизируемое избранное, избранное на
flat.html, заметки/папки/теги, экспорт избранного, лимит на количество.

## Архитектура

### 1. Хранилище — `static/favorites.js` (новый модуль)

Тонкая обёртка над `localStorage`, ключ `kvadstat:favorites:v1` — JSON-массив
числовых id квартир. Публичное API на `window.KV.fav`:

- `has(id) -> bool`
- `toggle(id) -> bool` (возвращает новое состояние: true = теперь в избранном)
- `add(id)`, `remove(id)`
- `list() -> number[]`
- `count() -> number`
- `onChange(cb)` — подписка; вызывается при локальном toggle и при событии
  `storage` (изменение из другой вкладки браузера).

Защищённый парсинг (try/catch, как у существующих localStorage-guards):
битый JSON или отсутствие localStorage → пустой список, страница не падает.
Id нормализуются к Number и фильтруются по Number.isFinite — мусор в хранилище
не ломает выборку.

Загружается в `<head>` обеих страниц после common.js (чтобы KV уже существовал),
с `defer` — как остальные локальные скрипты.

### 2. Кнопка в панели деталей (index.html)

В шапку панели (`.detail-head`, рядом с крестиком закрытия) добавляется
toggle-кнопка `#detailFavBtn`:
- «☆ В избранное» — когда не в избранном;
- «★ В избранном» — когда в избранном (акцентный цвет).

Кнопка в шапке (не в теле) — всегда видна независимо от прокрутки и попадает
в focus-trap панели. `openDetail(row)` запоминает `detailFlatId = row.id` и
проставляет состояние кнопки. Клик → `KV.fav.toggle(detailFlatId)` → обновляет
подпись кнопки и (через общий `refreshFavUI`) подсветку строки и счётчик вкладки.

### 3. Подсветка в «Все объекты»

Общий `rowFormatter` (используется и таблицей объектов, и таблицей избранного)
добавляет класс `is-fav` строкам, чей id в избранном. CSS — мягкая нейтральная
подложка (в духе сделанной ранее бледной заливки колонки «Класс»), без яркости.
При toggle конкретная строка переформатируется (`table.getRow(id).reformat()`),
полный redraw не требуется.

### 4. Вкладка «⭐ Избранное (N)»

Четвёртая кнопка в `nav.tabs` со счётчиком N (`#favCount`), новая секция
`#tab-favorites`. Контент — отдельный экземпляр Tabulator (`favTable`),
создаётся лениво при первом открытии вкладки, переиспользует определения колонок
(`columns.map(c => ({...c}))` — мелкий клон, чтобы Tabulator не делил внутреннее
состояние колонок между инстансами) и тот же `rowFormatter` и rowClick→openDetail.

Данные:
- **активные**: фильтр уже загруженного `allRows` по id из избранного —
  без сетевого запроса;
- кнопка-тумблер **«Показать снятые с продажи»** (`#favShowGone`): вычисляет
  недостающие id (избранные, которых нет в allRows), одним запросом
  `/kvadstat/fav_flats.json?ids=<json>&_shape=array` тянет их последний срез,
  помечает `_gone = true`, мёржит в таблицу. Снятые строки получают класс
  `is-gone` (приглушение) и бейдж «снято · <дата_среза>».
- **пусто**: «Пока ничего не добавлено. Откройте квартиру и нажмите ★».

Переключение в эту вкладку и обратно, а также любой toggle при открытой вкладке,
перестраивают данные favTable.

### 5. Backend — canned-query `fav_flats` (metadata.yml)

`default_allow_sql off` блокирует произвольный `?sql=`, но предопределённые
canned-queries работают. Добавляется query `fav_flats`, повторяющий колоночную
форму today_all (русские имена), но без ограничения «текущий срез»: для каждого
запрошенного id берётся его последний снапшот.

```sql
WITH want AS (SELECT value AS id FROM json_each(:ids)),
latest AS (
    SELECT s.flat_id, MAX(s.scan_date) AS scan_date
    FROM snapshots s JOIN want w ON w.id = s.flat_id
    GROUP BY s.flat_id
)
SELECT f.id AS id, b.developer AS застройщик, ... s.scan_date AS дата_среза,
       f.block_id AS block_id
FROM flats f
JOIN latest l ON l.flat_id = f.id
JOIN snapshots s ON s.flat_id = f.id AND s.scan_date = l.scan_date
LEFT JOIN blocks b ON b.id = f.block_id
```

`:ids` передаётся как JSON-массив строкой (`?ids=[123,456]`, url-encoded);
`json_each` разворачивает его в строки. Колонки именуются ровно как в today_all,
чтобы строки без переформатирования ложились в тот же Tabulator. Поля, которых
нет дёшево (этажей_всего по корпусу), берутся из `b.floors_max` —
для приглушённой архивной строки точность некритична.

## Поток данных

```
localStorage[kvadstat:favorites:v1]  ──┐
                                       ├─► KV.fav (has/toggle/list/onChange)
storage-событие (другая вкладка) ──────┘            │
                                                    ▼
                          refreshFavUI(): счётчик N, reformat строки,
                                          подпись кнопки, перестроение favTable
                                                    │
        активные: allRows ∩ favs ──────────────────┤
        снятые (по кнопке): /kvadstat/fav_flats.json?ids=… ─► merge (_gone)
```

## Обработка ошибок

- localStorage недоступен/битый → пустой список, без падения.
- `fav_flats` вернул ошибку/пусто → показываем только активные, рядом с кнопкой
  «Показать снятые» — короткое «не удалось загрузить снятые», не валим вкладку.
- id в избранном, которого нет ни в allRows, ни в БД → просто отсутствует
  в выдаче (id остаётся в localStorage — вернётся, если квартира появится).

## Тестирование

- pytest: тест canned-query `fav_flats` на временной БД (корректные русские
  колонки, выборка по списку id, последний срез).
- playwright (UI): добавить из панели → счётчик и вкладка показывают квартиру →
  подсветка в «Все объекты» → перезагрузка сохраняет → удаление убирает строку.
- Гейты: ruff / mypy / pytest — все зелёные. Живая проверка playwright на проде
  после деплоя (включая reload datasette для новой canned-query).

## Деплой

`deploy/deploy.sh` синхронизирует код + статику + metadata.yml. Новая
canned-query требует **рестарта datasette** (metadata читается при старте) —
убедиться, что deploy перезапускает unit при изменении metadata.yml.
